# Topic Selection

Use this file when the user gives no premise.

## Selection Ladder

1. Start from one friction that can be dramatized in a single sitting.
2. Give one protagonist a simple readable desire: get home, hide something, delay something, return something, avoid recognition, keep a promise, keep a lie alive.
3. Add one concrete blocker: another person, a system, weather, timing, money, obligation, or an object.
4. Pick one anchor object that can recur across the story.
5. Choose an ending mode before drafting so the story does not sprawl.

## Reliable Premise Families

- A routine errand that exposes a private compromise.
- A message that is read too late, sent to the wrong person, or never sent.
- A small workplace task that turns morally or emotionally expensive.
- A family request that sounds practical but carries old debt.
- A stranger encounter that forces the protagonist to recognize something they were dodging.
- An ordinary object that arrives where it should not.
- A lightly uncanny event that intensifies an existing pressure instead of replacing it.

## Filters

- Prefer premises that can stay inside `1-3` locations.
- Prefer stories that need at most `1-3` consequential characters.
- Avoid premises that depend on heavy exposition, legal procedure, or large-scale history unless the user explicitly asks for them.
- Avoid repeating the same engine, setting, or ending mode as the last few issues when that history is available.
