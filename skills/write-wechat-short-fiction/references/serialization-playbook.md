# Serialization Playbook

Use this file when one story needs to run across multiple days.

## Trigger Conditions

Switch to serial mode when any of these are true:

- The premise has multiple major turns that would be flattened in one post.
- The story needs more than one major turn to feel earned.
- Cutting to one post would force summary instead of scene.
- The user explicitly asks for a multi-part arc or 连载.

## Arc Planning

- Default to `2-4` episodes.
- Give each episode one dominant movement: setup, escalation, complication, reveal, return, or cost.
- Keep the whole serial anchored to the same central pressure. Do not let later episodes become a different story.
- Prefer one stable protagonist and no more than `1-3` consequential characters across the arc unless the user asks for a broader cast.

## Episode Contract

- Usually keep each installment within `1000-1800` Chinese characters, but let dramatic rhythm win over arbitrary precision.
- Make each installment readable on its own even if it continues tomorrow.
- Resolve one local movement per episode.
- Leave forward pressure for the next installment, but avoid fake cliffhangers that exist only to force a click.
- Re-introduce only the facts the reader needs right now. Do not recap the previous episode at length.

## Title Rule

- Keep the archive pattern and add the part marker inside the final title segment.
- Preferred format: `《短篇小说｜YYYY.MM.DD｜篇名（1）》`, `《短篇小说｜YYYY.MM.DD｜篇名（2）》`.
- Use `（上）/（中）/（下）` only when the user explicitly prefers that style.

## Failure Modes

- If episode 1 only feels like a prologue, move one concrete decision or consequence into it.
- If later episodes repeat the same beat with different decoration, compress the arc.
- If the serial needs more than `4` episodes, reconsider the premise or cut subplots before continuing.
